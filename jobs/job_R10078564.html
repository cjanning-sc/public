Cyber/ Software Security Assurance Engineer (Satellite/Space Systems)
R10078564
Days (United States of America)

This is an early career position for an experienced Cybersecurity DevSecOps engineer in a 100% Agile development environment focused on ground segment and mission application software development.  Cyber Systems Engineers are focused on software security assurance activities, such as Static and Dynamic security scanning of application software; advocating for Secure Coding best practices; application layer hardening; and environment & infrastructure hardening per CM Standards (e.g., DISA STIGs); infrastructure as code (Ansible scripting, deployment/re-configuration automation). 

The Cyber/Software Systems Security Assurance Engineer works as part of a team contributing to the mission by following instructions from leads/mentors, and identifying opportunities to develop technical leadership skills. 

Additional Software Security Assurance Responsibilities include:

  • Working in an Agile engineering environment, where the Cyber Systems Engineer may assist identifying and prioritizing security requirements; triage of Static Code Analysis (SCA) tool findings (e.g. Fortify) and assist in prioritizing the findings as technical debt in the Software Development LifeCycle (SwDLC) backlog.
  • Advising software development scrum teams on secure coding practices, security-focused engineering trade studies, and other security best practices.
  • Conducting system vulnerability scanning, remediation and patch management activities on Windows and Red Hat operating systems and various COTS/GOTS applications, including those within virtualized and/or cloud environments. 
  • Documenting Standard Operating Procedures (SOPs), and when needed, performing software patch installation, other flaw remediation, antivirus updates, and continuous monitoring (ConMon) activities.
  • Ensuring systems are operated, maintained, and disposed of in accordance with security policies and procedures as outlined in the system security authorization package. 
  • Developing standard Risk Management Framework (RMF) artifacts, such as System Security Plan (SSP), Risk Assessment Report (RAR), Security Controls Traceability Matrix (SCTM), and Plans of Action & Milestones (POA&Ms), Contingency Plans, Incident Response Plans, and more.

Basic Qualifications:

  • Bachelors Degree in STEM field
  • 2 years IT security experience in any industry
  • Must have U.S. Citizenship and suitability for submission for TS clearance
  • Any DoD 8570 IT Security certification, such as Security+
  • Exposure to NIST 800-37 security controls and the Systems Engineering requirements analysis, decomposition, and analysis process

Preferred Qualifications:

  • Bachelors Degree in Electrical, Computer, or Aerospace Engineering
  • 2+ years experience in the Defense Aerospace Industry
  • CISSP or CSSLP Certification (Associate version)
  • Active DoD security clearance
  • Demonstrated knowledge of Systems Engineering review processes, milestones and SDLC
  • Demonstrated interest in the securing of complex, embedded systems
