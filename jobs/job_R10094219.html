ACME Engineering Space Systems – Launch and Missile Defense Systems has an exciting career opportunity for a Principal Systems Engineer  - Interface to join our team of qualified, diverse individuals. This position will be located out of Indianapolis, IN.

Position Overview:

C2BMC External or Internal Interface Engineer conducting interface definition and revision to incorporate new system capabilities and spiral development changes that affect the C2BMC interface. Primary responsibility will be developing and updating XML schemas (.xsd files) to support multiple Interface Control Documents (ICDs). Will work with multiple ICD authors and participate in multiple technical working groups to apply knowledge of the system domain and interface exchange requirements. Must have working knowledge of XML and proficiency in XML Spy.

Essential Functions:

  • Successful candidate will attend interface working group meetings with MDA customer and representatives of assigned external element engineering organization as well as support to program integration and test related efforts for the associated interface.
  • Successful candidate may also be assigned to support: other interface engineers in developing ICDs for other BMDS Elements; XML schema development; requirements engineering, use case development, and logical architecture modeling tasks as SE department tasking dictates.
  • Successful candidate will support spiral leadership to assure interface artifacts and system assessment tasks required for the assigned sensor interface are completed when required.

Basic Qualifications:

If applicable, please note your updated security clearance and IAT/relevant certifications on your resume.

  • An active Secret clearance is required.
  • 5 Years with Bachelor’s degree in a related field; 3 years’ experience with a Master’s degree.
  • Demonstrated Systems Engineering skills and experience with interface engineering, to include: message definition and processing rules; message formatting (e.g. XML, VMF, et al); physical/network specifications and models, to include the ISO OSI model; physical/network topologies and devices; network protocols (e.g. TCP/IP, UDP, et al).
  • In addition, requirements engineering, to include experience working with IBM Rational's Dynamic Object Oriented Requirements System (DOORS).
  • Mid-Career experience performing pre-PDR systems engineering for DoD Information Systems.
  • Must be able to function as part of an experienced team in developing systems engineering products and artifacts while adhering to the program Architecture and Systems Engineering Management Plan (ASEMP) and other processes.

Preferred Qualifications:

  • Desired skills include: requirements engineering, use case development, logical architecture modeling, MBSE modeling, to include: behavioral modeling (use cases, activity diagrams, sequence diagrams, statechart diagrams, etc.); structural modeling (structural composition diagrams, interface diagrams, conceptual object diagrams, etc.); et al., SysML, BMDS domain experience, RMF-based cyber security ,and project planning utilizing MS Project, and Earned Value Management techniques and tools.
  • Experience with Agile or Iterative Development Methods.
  • Past experience with LRDR or other MDS sensor elements is a significant plus.
  • Experience with team leadership, task tracking, and issue resolution coordination is a significant plus.
  • Experience with Agile/Iterative development methods is also a significant plus.
  • Missile Defense Agency (MDA) sensor domain experience a plus.
  • Specific interface experience with the MDA elements or sensors is a significant plus.
